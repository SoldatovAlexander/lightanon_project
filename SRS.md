Software Requirements Specification (SRS)

## Project: LightAnon (v0.1 MVP)

**Дата:** 3 марта 2026
**Статус:** Beta
**Тип:** Python Library

---

## 1. Введение
**LightAnon** — это легковесная библиотека на Python для анонимизации (обезличивания) структурированных данных.
**Основная цель:** Предоставить ML-инженерам инструмент для подготовки датасетов, который обеспечивает соответствие 152-ФЗ (Приказ Роскомнадзора №996), сохраняя при этом статистическую полезность данных (Utility) для обучения моделей.

---

## 2. Функциональные требования (FR)

### 2.1. Core Engine (Ядро)
* **FR-01:** Библиотека должна принимать на вход данные в формате `pandas.DataFrame` и `dict` (JSON-like).
* **FR-02:** Обработка должна происходить по конвейерному принципу (Pipeline): `Input -> Rules Application -> Output`.
* **FR-03:** Конфигурация правил анонимизации должна передаваться через Python Dictionary или YAML-файл.
* **FR-04:** Библиотека должна поддерживать механизм обработки ошибок: если правило падает на одной колонке, процесс не должен прерываться (fail-safe), ошибка логируется в отчет.

### 2.2. Методы анонимизации (General Rules)
Библиотека должна реализовывать следующие методы трансформации:

* **FR-05 (Hashing):** Детерминированное хэширование (SHA-256) с использованием "соли" (Salt).
    * *Цель:* Возможность JOIN-операций между таблицами по ID.
    * *Юридическое обоснование:* Метод введения идентификаторов (Приказ №996, п.1).
* **FR-06 (Masking):** Частичное скрытие символов (пример: `i*****@mail.ru`).
    * *Параметры:* Количество видимых символов, позиция (start/end).
* **FR-07 (Generalization/Binning):** Преобразование чисел в интервалы (пример: 23 -> "20-25").
    * *Параметры:* Шаг интервала (step).
* **FR-08 (Gaussian Noise):** Добавление аддитивного шума из нормального распределения.
    * *Параметры:* Стандартное отклонение (std) или процент от значения.
    * *Цель:* Сохранение среднего значения по выборке.
* **FR-09 (Date Shifting):** Сдвиг дат на случайное количество дней в пределах заданного окна.
    * *Цель:* Сохранение интервалов между событиями.

### 2.3. Специализированные методы (Financial Module)
Для работы с финансовыми данными (FinTech):

* **FR-10 (Multiplicative Noise):** Умножение на случайный коэффициент (пример: `Value * random(0.95, 1.05)`).
    * *Цель:* Сохранение трендов для транзакций разного масштаба.
* **FR-11 (Top-Coding / Winsorization):** Ограничение выбросов. Все значения выше перцентиля (например, 99%) заменяются на значение этого перцентиля.
    * *Цель:* Защита от атак пере-идентификации по уникальным крупным суммам ("киты").
